# QingToolbox 0.1.0-alpha Preview

## 版本定位

这是 QingToolbox 的首个公开 Preview，面向愿意测试 Windows 桌面模块宿主、反馈兼容性和交互问题的用户。它不是稳定版，也不建议用于企业关键生产环境。

发布资产仅包含 QingToolbox 宿主，不内置 TextTools、PowerGuard 或其他具体模块。模块通过独立的 `.qmod` 文件导入。

## 主要功能

- 轻量 WPF Shell、模块列表、模块详情和独立模块窗口。
- 只读取清单的模块发现与 `.qmod` 导入；Import 和 Refresh 不加载模块 DLL。
- 用户主动执行 Load、Activate、Deactivate 和 Unload。
- 基于 collectible `AssemblyLoadContext` 的进程内模块卸载基础设施。
- 简体中文和英文宿主本地化，以及模块本地化契约。
- 当前用户范围的单实例、Windows 登录自启动和启动显示模式。
- 桌面悬浮标、Windows 通知区域恢复图标和可配置的主窗口关闭行为。

## 模块安全模型

`.qmod` 导入后默认保持 `NotLoaded`。通常需要用户手动 Load 和 Activate；Refresh 和 Import 始终只发现模块。

用户可以为单个模块明确开启“随工具箱启动”。宿主会保存经过验证的完整模块载荷指纹，并在下次启动时重新计算。只有完全匹配时才执行 Load，并在授权要求激活时执行 Activate。模块内容变化会使旧授权失效，必须重新确认。Windows 登录自启动与模块“随工具箱启动”是两项独立设置。

当前模块在 QingToolbox 宿主进程内、以当前用户权限运行。权限列表是声明和用户告知，不是系统级沙箱；载荷指纹用于启动授权一致性，也不是代码签名或安全隔离。只应导入可信来源的模块。

Module API 状态为 **Experimental**。独立 NuGet SDK、稳定 API Version 和 0.x 二进制兼容承诺尚未发布，详情见 [`../sdk/README.md`](../sdk/README.md)。

## 安装版与便携版

安装版：`QingToolbox-0.1.0-alpha-win-x64-setup.exe`

- 当前用户安装，不要求管理员权限。
- 提供开始菜单入口、卸载项和可选桌面快捷方式。
- 默认不会开启 Windows 登录自启动，也不会导入模块。
- 未代码签名，Windows SmartScreen 可能显示“未知发布者”。

便携版：`QingToolbox-0.1.0-alpha-win-x64.zip`

- 解压后直接运行 `QingToolbox.Shell.exe`。
- 需要 Windows x64 和 .NET 10 Windows Desktop Runtime。
- 与安装版一样，只包含宿主，不包含具体模块或用户设置。

## 首次使用流程

1. 安装或解压 QingToolbox 并启动。
2. 打开“模块”，选择“导入模块”。
3. 选择可信的 `.qmod` 文件。导入完成后模块仍为未加载。
4. 主动点击 Load；需要运行服务或 UI 时再 Activate/Open。
5. 不再使用时可 Deactivate 或 Unload。

## 通知区域和关闭行为

首次关闭主窗口时，QingToolbox 会询问：最小化到通知区域，或完全退出。设置页可随时修改选择，也可恢复“下次重新询问”。

通知区域图标左键恢复主窗口；右键菜单提供打开 QingToolbox、切换悬浮标、打开设置和退出 QingToolbox。关闭到通知区域不会卸载或停用模块。真正退出会停止接受二次启动激活、关闭模块窗口、卸载模块并移除通知区域图标。

应用使用显式 Shutdown 管理后台生命周期，不依赖最后一个窗口关闭。正常用户不需要运行 `stop-qingtoolbox.bat`。

## 开机启动与模块恢复

Windows 登录自启动只控制宿主是否在登录后启动；启动显示模式可选择主窗口、任务栏最小化或悬浮标。它不会自动授权任何模块。

只有明确开启“随工具箱启动”的模块才会进入恢复流程。宿主在恢复前及 Load 前验证完整载荷；验证失败、文件变化或模块缺失都会跳过执行并要求重新确认。

## 已知限制

- 当前仅支持 Windows x64，并基于 .NET 10 Windows Desktop。
- 程序和安装器尚未代码签名，SmartScreen 可能提示未知发布者。
- Module API 仍为 Experimental，0.x 兼容性不保证。
- 第三方模块运行在宿主进程内，不具备进程或系统级沙箱隔离。
- 权限列表目前是声明和用户告知，不是强制沙箱。
- 模块开发仍需使用仓库中的 `QingToolbox.Abstractions` 和 `modules` 分支模板。
- 自动更新尚未实现。
- 不内置在线模块商店。
- `.qmod` 包签名和稳定的第三方 SDK 尚未发布。
- Windows 11 Snap、混合 DPI、多显示器、Explorer 重启及关机/注销路径仍需要更多实体环境验证。

## 校验 SHA256

每个二进制资产旁都有同名 `.sha256` 文件。PowerShell 校验示例：

```powershell
Get-FileHash .\QingToolbox-0.1.0-alpha-win-x64-setup.exe -Algorithm SHA256
Get-FileHash .\QingToolbox-0.1.0-alpha-win-x64.zip -Algorithm SHA256
```

将输出与 `.sha256` 文件以及 `QingToolbox-0.1.0-alpha-win-x64.manifest.json` 比较。Manifest 同时记录源码 commit、资产大小和 SHA256。
