# QingToolbox 0.1.0-alpha Preview

这是面向少量测试用户的预览版本，不是正式稳定版，也不代表生产环境可用。

## 安装版

运行 `QingToolbox-0.1.0-alpha-win-x64-setup.exe`，按向导完成安装。安装器是
self-contained、仅当前用户安装，不需要管理员权限，默认路径为：

```text
%LOCALAPPDATA%\Programs\QingToolbox
```

安装程序只包含 QingToolbox 宿主，不内置 TextTools、ScreenPin、
WindowTopmost 或其他具体模块。开始菜单快捷方式会自动创建，桌面快捷方式
默认不选择。安装器跟随 Windows UI 语言提供英文或简体中文，自定义任务、
卸载快捷方式和安装结束启动文字也会使用相应语言。

下载后可使用 PowerShell 校验安装器：

```powershell
Get-FileHash .\QingToolbox-0.1.0-alpha-win-x64-setup.exe -Algorithm SHA256
```

结果应与 Release 中的 `.sha256` 文件一致。

## 便携版

1. 下载 `QingToolbox-0.1.0-alpha-win-x64.zip`。
2. 校验发布页提供的 SHA256。
3. 解压到普通用户可读目录。
4. 安装 .NET 10 Desktop Runtime。
5. 运行 `QingToolbox.Shell.exe`。

便携版不写安装记录，可直接删除解压目录。

## 卸载与数据保留

可从“Windows 设置 → 应用 → 已安装的应用 → QingToolbox → 卸载”，或使用
开始菜单中与安装语言一致的卸载入口。卸载会删除主程序、
快捷方式和安装记录，但默认保留用户导入模块、模块数据和设置。

需要完整清理时，在卸载后手动删除：

```text
%LOCALAPPDATA%\QingToolbox
%APPDATA%\QingToolbox
```

安装器和程序目前没有代码签名，Windows SmartScreen 可能显示未知发布者警告。

## 自动发布验证

Windows CI 会构建便携 ZIP 和 self-contained 安装器，重新计算两者的 SHA256，
运行模块 Smoke Test，并在隔离用户目录中完成静默安装—卸载往返测试。测试会
确认卸载保留用户模块、模块数据和 `settings.json` 哨兵。CI artifacts 仅用于
Preview 验证，不会自动创建 GitHub Release 或 tag。正式应用图标仍待提供。

CI 不能替代 UAC、双语向导视觉、SmartScreen、开始菜单、桌面快捷方式交互、
GUI 启动和 DPI 表现的人工检查，以下手动清单因此继续保留。

为保证可复现性，CI 固定校验官方简体中文 Inno Setup 翻译文件的 SHA256，避免
重复执行已经通过的构建与 Smoke Test，并通过 `/NOICONS` 隔离 Roundtrip。
失败时只上传 `install.log`、`uninstall.log` 和文本诊断，不上传用户数据或载荷。
发布者可运行 `./scripts/verify-preview-assets.ps1` 独立复核两个发布资产。

版本和资产名称统一从 `Directory.Build.props` 派生。CI 生成
`QingToolbox-0.1.0-alpha-win-x64.manifest.json`，其中记录完整源码 commit、
两个资产的文件名、大小和重新计算的 SHA256。官方 Actions 及中文 Inno 翻译
都固定到不可变 commit；此可追溯清单仍不会触发自动 Release 或 tag。

## 模块目录

- 程序目录下的 `Modules`：开发或随程序提供的模块，优先发现。
- `%LOCALAPPDATA%\QingToolbox\Modules`：用户导入的 `.qmod` 模块。
- `%APPDATA%\QingToolbox\Data`：模块运行数据。
- `%APPDATA%\QingToolbox\settings.json`：用户设置。

启动和“刷新模块”只扫描 `module.json`，不会加载模块 DLL。模块必须由用户
手动点击“加载”。

## 导入 `.qmod`

在“模块”页面点击“导入模块”，选择一个 `.qmod` 文件。导入器先在临时目录
解压并验证清单和入口 DLL，再移动到用户模块目录。同 ID 模块不会被覆盖。

`.qmod` 尚未提供签名校验。模块加载后拥有当前用户权限，请只导入可信来源。
格式说明参见 [`../QMOD_FORMAT.md`](../QMOD_FORMAT.md)。

## Preview 候选门禁

发布候选必须先运行 `./scripts/build-preview-release-candidate.ps1`，确认源码工作区干净、
`toolbox` 与 `origin/toolbox` 同步、完整构建与安装往返测试通过，并核对 schema v2
manifest 的仓库、commit、clean-source 声明和资产哈希。详细交接步骤见
[`../PREVIEW_RELEASE_PROCESS.md`](../PREVIEW_RELEASE_PROCESS.md)。该门禁不会自动创建
Release 或 tag。

## Known Issues

- 这是 Preview，不是稳定版本。
- 二进制文件和 `.qmod` 包均未签名。
- 没有自动更新。
- ScreenPin 的几何、DPI 与缩放体验仍需完善。
- 模块覆盖升级、依赖解析和权限隔离尚未实现。
