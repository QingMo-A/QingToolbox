# QingToolbox 0.1.0-alpha Preview

这是面向少量测试用户的预览版本，不是正式稳定版，也不代表生产环境可用。

## 如何运行

1. 下载 `QingToolbox-0.1.0-alpha-win-x64.zip`。
2. 校验发布页提供的 SHA256。
3. 解压到普通用户可读目录。
4. 运行 `QingToolbox.Shell.exe`。

默认 zip 是 framework-dependent 发布，需要安装 .NET 10 Desktop Runtime。

本 Preview 没有安装程序，也没有代码签名。Windows SmartScreen 可能显示
未知发布者警告。

## 模块目录

- 程序目录下的 `Modules`：开发或随程序提供的模块，优先发现。
- `%LOCALAPPDATA%\QingToolbox\Modules`：用户导入的 `.qmod` 模块。
- `%APPDATA%\QingToolbox\Data`：模块运行数据。
- `%APPDATA%\QingToolbox\settings.json`：用户设置。

启动和“刷新模块”只扫描 `module.json`，不会加载模块 DLL。模块必须由用户
手动点击“加载”。

## 导入 `.qmod`

在“模块”页面点击“导入模块”，选择一个 `.qmod` 文件。导入器先在临时目录
解压并验证清单和入口 DLL，再移动到用户模块目录。同 ID 模块不会被覆盖，
Preview 阶段需要手动删除旧模块后重新导入。

`.qmod` 尚未提供签名校验。模块加载后拥有当前用户权限，请只导入可信来源的模块。
格式说明参见 [`../QMOD_FORMAT.md`](../QMOD_FORMAT.md)。

## Known Issues

- 这是 Preview，不是稳定版本。
- 二进制文件和 `.qmod` 包均未签名。
- 没有安装器、卸载器和自动更新。
- ScreenPin 的几何、DPI 与缩放体验仍需完善。
- 模块覆盖升级、依赖解析和权限隔离尚未实现。

## 发布前手动测试清单

- [ ] 启动 Shell。
- [ ] 确认 Refresh Modules 不加载 DLL。
- [ ] Load TextTools。
- [ ] Open TextTools。
- [ ] Deactivate TextTools。
- [ ] Unload TextTools。
- [ ] 在 zh-CN 与 en-US 之间切换语言。
- [ ] 导入一个有效 `.qmod` 并在刷新后显示。
- [ ] 导入损坏的 `.qmod`，确认失败且无残留目录。
- [ ] 导入包含 `../` 的恶意 `.qmod`，确认被拒绝。
- [ ] 再次导入同 ID 模块，确认拒绝覆盖。
- [ ] 退出程序，确认已加载模块能够卸载。
