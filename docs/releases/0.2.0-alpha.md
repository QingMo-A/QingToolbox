# QingToolbox 0.2.0-alpha — Preview 2

**Reliable Startup & Module Lifecycle / 可靠启动与模块生命周期**

Preview 2 introduces least-privilege Task Scheduler login startup, visible presentation before module discovery, and startup health, test, repair, and recovery diagnostics. Production, Development, and ModuleTest data remain isolated.

The module workflow can detect official updates and download packages with size and SHA256 verification. Downloaded `.qmod` files remain staged only: QingToolbox does not automatically extract, install, replace, or roll back modules. User-installed modules can be removed safely while module data is preserved, and their installation directory can be opened from module details.

Host upgrades are performed by running a newer QingToolbox installer over the existing installation. Preview 2 preserves settings, user modules, module data, caches, and startup authorizations. The installer reuses the existing application identity and directory, supports same-version repair, records its installed version, and rejects replacement of a newer installed version.

The portable ZIP is **framework-dependent** and requires the .NET 10 Desktop Runtime. It contains no concrete tool modules. The installer is self-contained.

Known limitations: module updates are not installed automatically; the already-published Preview 1 installer cannot retroactively reject downgrades; real sign-in startup, Explorer recovery, multi-display/DPI behavior, and in-place upgrades on representative user machines still require manual verification.
